BeeSecure – KeyLock

Diagram blokowy systemu · przepływ danych i warstwy

Architektura logiczna: użytkownicy korzystają z panelu webowego lub bezpośrednio z zamka; centrum to serwer i API; w obiekcie gateway łączy sieć lokalną z siecią ZigBee urządzeń.

Legenda

  • HTTPS — dostęp do panelu i API z przeglądarki.
  • TLS / MQTT / REST — kanał z serwera do gateway w obiekcie (model zależny od wdrożenia).
  • ZigBee — sieć bezprzewodowa między gateway a zamkami.
  • Kody lokalne — gość może otworzyć zamek kodem bez udziału personelu; synchronizacja seed może działać także przy ograniczonej łączności.
